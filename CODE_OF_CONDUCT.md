# Code Of Conduct

Be respectful, precise, and source-oriented.

Contributions should focus on official public records, careful caveats, and reproducible local tooling. Do not use the project to harass people or make unsupported public accusations.

## Expected Behavior

- Treat contributors and source maintainers with respect.
- Keep claims tied to official source records and checked times.
- Flag uncertainty clearly.
- Minimize unnecessary personal data in examples, issues, and tests.

## Unacceptable Behavior

- Harassment or personal attacks.
- Publishing unnecessary personal data.
- Using the project to make unsupported accusations.
- Asking contributors to bypass access controls.

## Reporting

Open an issue for conduct concerns that can be discussed publicly. For sensitive concerns, request a private maintainer contact path without posting sensitive details.
