# Security Policy

Report security concerns by opening a private security advisory on the public repository once advisories are enabled. If advisories are not available, open a minimal issue asking maintainers to enable private disclosure without posting sensitive details.

Do not include sensitive personal data in reports unless it is necessary to explain the issue.

OpenTrade Registry should not bypass CAPTCHAs, login walls, paywalls, or technical access controls.

## Supported Versions

OpenTrade Registry is pre-1.0. Security fixes will target the latest `main` branch until versioned releases begin.

## Security-Sensitive Areas

- Source fetching and future network sync
- CLI file path handling
- Raw public-record preservation
- Generated exports
- Package publishing and release provenance
